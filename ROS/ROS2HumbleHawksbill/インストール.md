以降のチュートリアルで使用するコマンドは、お使いのオペレーティングシステムのバイナリパッケージ（Linuxの場合はdebパッケージ）のインストールガイドに従っていることを前提としています。ソースからビルドした場合でも、以降のチュートリアルの手順に従うことは可能ですが、セットアップファイルへのパスが異なる場合があります。また、ソースからインストールした場合は、（初心者向けチュートリアルで頻繁に使用される）sudo apt install ros-<distro>-<package_name>コマンドは使用できません。
ロケールを設定
UTF-8をサポートしているロケールを使用していることを確認してください。Dockerコンテナなどの最小限の環境の場合は、ロケールがPOSIXのような最小限のものになっている可能性があります。以下の設定でテストしています。ただし、UTF-8をサポートする別のロケールを使用している場合でも、問題なく動作するはずです。

```bash
locale  # check for UTF-8

sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

locale  # verify settings
```


セットアップソース
ROS 2 apt リポジトリをシステムに追加する必要があります。
まず、[Ubuntu Universe リポジトリ](https://help.ubuntu.com/community/Repositories/Ubuntu)が有効になっていることを確認します。

```bash
sudo apt install software-properties-common
sudo add-apt-repository universe
```


ros [-apt-source](https://github.com/ros-infrastructure/ros-apt-source/)パッケージは、さまざまな ROS リポジトリのキーと apt ソース構成を提供します。
ros2-apt-source パッケージをインストールすると、システムの ROS 2 リポジトリが設定されます。このパッケージの新しいバージョンが ROS リポジトリにリリースされると、リポジトリ設定が自動的に更新されます。

```bash
sudo apt update && sudo apt install curl -y
export ROS_APT_SOURCE_VERSION=$(curl -s https://api.github.com/repos/ros-infrastructure/ros-apt-source/releases/latest | grep -F "tag_name" | awk -F\" '{print $4}')
curl -L -o /tmp/ros2-apt-source.deb "https://github.com/ros-infrastructure/ros-apt-source/releases/download/${ROS_APT_SOURCE_VERSION}/ros2-apt-source_${ROS_APT_SOURCE_VERSION}.$(. /etc/os-release && echo ${UBUNTU_CODENAME:-${VERSION_CODENAME}})_all.deb"
sudo dpkg -i /tmp/ros2-apt-source.deb
```


ROSパッケージのインストール
リポジトリを設定した後、apt リポジトリ キャッシュを更新します。
```bash
sudo apt update
```

ROS 2パッケージは、頻繁に更新されるUbuntuシステム上に構築されています。新しいパッケージをインストールする前に、システムが最新であることを確認することをお勧めします。
```bash
sudo apt upgrade
```

Ubuntu 22.04 の初期アップデートのため、 ROS 2 をインストールする前にsystemd、および関連パッケージを更新することが重要です。アップグレードせずに新規にインストールしたシステムに ROS 2 の依存関係をインストールすると、重要なudevシステム パッケージが削除される可能性があります。
詳細については、[ros2/ros2#1272](https://github.com/ros2/ros2/issues/1272)および[Launchpad #1974196](https://bugs.launchpad.net/ubuntu/+source/systemd/+bug/1974196)を参照してください。
デスクトップ インストール (推奨): ROS、RViz、デモ、チュートリアル。
```bash
sudo apt install ros-humble-desktop
```

開発ツール: ROS パッケージを構築するためのコンパイラおよびその他のツール
```bash
sudo apt install ros-dev-tools
```

アンインストール
ROS 2 をアンインストールする必要がある場合、またはバイナリからインストールした後にソースベースのインストールに切り替える必要がある場合は、次のコマンドを実行します。
```bash
sudo apt remove '~nros-humble-*' && sudo apt autoremove
```

リポジトリを削除することもできます。

```bash
sudo apt remove ros2-apt-source
sudo apt update
sudo apt autoremove
sudo apt upgrade # Consider upgrading for packages previously shadowed.
```
