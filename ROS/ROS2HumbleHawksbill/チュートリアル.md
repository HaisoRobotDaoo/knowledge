環境設定
正確なコマンドは、ROS 2 をインストールした場所によって異なります。問題が発生した場合は、ファイル パスがインストールにつながっていることを確認してください。
セットアップスクリプトのソース
新しいシェルを開くごとに、次のファイルをソースとして環境を設定します。
```bash
source /opt/ros/humble/setup.bash
```

または、永続化したい場合は~/.bashrcに書き込みます。
```bash
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
```

セットアップファイルをソース（読み込み）しないと、ROS 2コマンドにアクセスしたり、ROS 2パッケージを検索したり使用したりできなくなります。つまり、ROS 2を使用することはできません。
環境変数の確認
ROS 2セットアップファイルをソースすると、ROS 2の操作に必要ないくつかの環境変数が設定されます。
ROS 2 パッケージの検索や使用で問題が発生した場合は、次のコマンドを使用して環境が適切に設定されていることを確認してください。
```bash
printenv | grep -i ROS
```

出力例

```bash
ROS_VERSION=2
ROS_PYTHON_VERSION=3
ROS_DISTRO=humble
```


話し手・聞き手
上記ros-humble-desktopをインストールした場合は、いくつかの例を試すことができます。
1 つのターミナルで、セットアップ ファイルをソースし、C++を実行しますtalker。

```bash
source /opt/ros/humble/setup.bash
ros2 run demo_nodes_cpp talker
```


別のターミナルでセットアップ ファイルをソースし、Python を実行しますlistener。

```bash
source /opt/ros/humble/setup.bash
ros2 run demo_nodes_py listener
```


これで、C++ APIとPython APIの両方が正常に動作していることが確認できます。
