# インターフェースの移行

## インターフェース定義

ROS 2では、インターフェースの種類ごとにフォルダを厳格に分ける必要があります。
メッセージファイルは `.msg` で終わる必要があり、サブフォルダ `msg/` に配置する必要があります。
サービスファイルは `.srv` で終わる必要があり、サブフォルダ `srv/` に配置する必要があります。
アクションファイルは `.action` で終わる必要があり、サブフォルダ `action/` に配置する必要があります。

これらのファイルは、[ROSインターフェース定義](https://design.ros2.org/articles/legacy_interface_definition.html)に準拠するために更新する必要があるかもしれません。
一部のプリミティブ型は削除され、ROS 1で組み込み型であった `duration` と `time` 型は通常のメッセージ定義に置き換えられたため、[`builtin_interfaces`](https://github.com/ros2/rcl_interfaces/tree/humble/builtin_interfaces)パッケージから使用する必要があります。
また、一部の命名規則はROS 1よりも厳格になっています。

詳細は[概念記事](https://docs.ros.org/en/humble/Concepts/Basic/About-Interfaces.html)をご覧ください。


## インターフェース構築

ROS 2におけるインターフェースの構築方法は、ROS 1とは大きく異なります。
ROS 2では、インターフェースをビルドできるのは `CMakeLists.txt` を持つパッケージのみです。

> **Pythonユーザーへの注意:** 純粋なPythonパッケージで自作メッセージを使いたい場合、インターフェース専用の別のCMakeパッケージを作るのが「ベストプラクティス（最善策）」です。

詳細については、[カスタムインターフェースのチュートリアル](https://docs.ros.org/en/humble/Tutorials/Beginner-Client-Libraries/Custom-ROS2-Interfaces.html)をご覧ください。

## インターフェースパッケージの移行

### あなたの package.xml ：

インターフェースを生成するための「ジェネレーター」と、実行時に必要な「ランタイム」を定義します。

```xml
<buildtool_depend>rosidl_default_generators</buildtool_depend>
<exec_depend>rosidl_default_runtime</exec_depend>

<member_of_group>rosidl_interface_packages</member_of_group>
```

依存するメッセージパッケージごとに、次を追加します。

```xml
<depend>message_package</depend>
```

### あなたの CMakeLists.txt ：

C++17を有効にする。
依存するメッセージパッケージごとに、CMake 関数呼び出しを `generate_messages` から `rosidl_generate_interfaces` に置き換えます。

```cmake
# C++17を有効化
set(CMAKE_CXX_STANDARD 17)

# 必要なパッケージのロード
find_package(rosidl_default_generators REQUIRED)
find_package(<dependent_message_package> REQUIRED)

# インターフェースの生成（.msgファイルなどをすべて列挙）
rosidl_generate_interfaces(${PROJECT_NAME}
  "msg/<message_file_name>.msg"
  "srv/<service_file_name>.srv"
  DEPENDENCIES <dependent_message_package>
)
```

これにより、すべてのメッセージファイルとサービスファイルを列挙する `add_message_files` および `add_service_files` が置き換えられるため、これらは削除できます。
