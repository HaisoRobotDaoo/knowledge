リモートPC側（親）の設定
①chronyを入れる

```bash
sudo apt update
sudo apt install chrony
```


② 設定ファイルを開く
```bash
sudo nano /etc/chrony/chrony.conf
```

③ファイルの一番下に追加する（重要）

```text
# ローカルネットワークからの接続を許可
allow 192.168.0.0/16

# インターネットがなくても自分の時刻を正しいものとして配る
local stratum 10
```


④ chrony を再起動
```bash
sudo systemctl restart chrony
```


TurtleBot側（子）の設定
①chronyを入れる

```bash
sudo apt update
sudo apt install chrony
```


② 設定ファイルを開く
```bash
sudo nano /etc/chrony/chrony.conf
```

③余計な設定を無効化する
ファイル内の以下のような行を すべて # でコメントアウト

```text
pool ntp.ubuntu.com iburst
server xxx.xxx.xxx.xxx
```


④一番下に PCのIPアドレス を書く￥

```text
# リモートPCのIPアドレスを指定
server 192.168.1.10 iburst
```


 192.168.1.10 は あなたのPCのIP に置き換えてください

リモートPCのIPアドレスを調べる方法
```bash
ip a
```

表示例（重要な所だけ）

```text
2: wlan0: <BROADCAST,MULTICAST,UP,LOWER_UP>
    inet 192.168.1.23/24 brd 192.168.1.255 scope global dynamic
```


- inet の後ろ  
- 192.168.1.23 ← これがIPアドレス


④ chrony を再起動
```bash
sudo systemctl restart chrony
```


同期できているか確認
TurtleBot側で実行
```bash
chronyc sources
```


成功の見方
表の M列（左端） を見る

記号	意味
*	✅ 今まさに同期中（成功）
+	予備の同期先
?	❌ 繋がっていない

* が付いていればOK
